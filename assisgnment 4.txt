1. **Six phases of SDLC (Software Development Life Cycle):**

   1. **Planning:** Identify project goals, scope, budget, and timeline.
   2. **Requirements Analysis:** Gather and document the user's needs and system requirements.
   3. **Design:** Create the system architecture, database, and user interface design.
   4. **Development (Implementation):** Write the code and build the software.
   5. **Testing:** Check the software for bugs and verify it meets the requirements.
   6. **Deployment & Maintenance:** Release the software to users and provide updates, fixes, and support.

2. **Three differences between Agile and Waterfall:**

| Agile                                              | Waterfall                                                         |
| -------------------------------------------------- | ----------------------------------------------------------------- |
| Development is done in small iterations (sprints). | Development follows one fixed sequence of phases.                 |
| Requirements can change during the project.        | Requirements are usually fixed before development starts.         |
| Customer feedback is collected regularly.          | Customer feedback is mostly taken after the project is completed. |

3. **Three main Scrum roles:**

   * **Product Owner:** Defines and prioritizes the product backlog to maximize business value.
   * **Scrum Master:** Guides the Scrum process, removes obstacles, and helps the team follow Scrum practices.
   * **Development Team:** Designs, develops, tests, and delivers the product increment.

4. **What is a Sprint in Scrum?**

   * A **Sprint** is a fixed period during which the Scrum team completes a set of planned work to deliver a usable product increment.
   * **Typical duration:** **1 to 4 weeks** (most commonly **2 weeks**).

5. **Meaning of Jira board columns:**

   * **Backlog:** A list of tasks or user stories waiting to be planned.
   * **To Do:** Tasks that are planned but have not been started.
   * **In Progress:** Tasks that are currently being worked on.
   * **Done:** Tasks that have been completed successfully.
